# 马拉松个人赛历和推荐

## 工作流程

### 1. 用户提问 → 先回答
- **默认只输出对话框文字结论**，不要自动生成文件
- 文字结论中每条赛事必须包含以下关键信息：
  **赛事名称 / 认证级别 / 比赛日期(精确到日) / 报名日期 / 出签日期 / 距用户交通耗时 / 难度备注(🟢🟡🔴)**
- 对话结束后**主动询问**是否需要整理成报告

### 2. 报告 → 询问格式
生成报告前必须问用户：是否生成？选 HTML 还是 PNG？

### 3. 采集 → 缺字段必须补
- 从闹跑获取的日期如果是月份模糊（如"12月"），**必须搜最新新闻确认具体日**
- 每场赛事必须采集完整的关键字段（见 references/required_fields.md）
- 撞期检查：同省同日赛事必须标注

### 4. 认证查询
- 田协 A/B/C 类：搜"2026年马拉松赛事目录 田协"
- 世界田联标牌：搜"2026世界田联标牌 中国 马拉松"

### 5. 交通查询
必须从 12306/携程 查真实数据，禁止估算。
搜格式："出发城市到目标城市 高铁 最快"

### 6. 赛道备注
每条推荐必须附带备注：爬升数据、景点、特色、PB潜力。
搜索引擎搜 "赛事名 赛道 爬升"

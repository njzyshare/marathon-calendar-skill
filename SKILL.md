# 马拉松个人赛历和推荐

## 跨工具兼容说明
本技能遵循标准 SKILL.md 格式，兼容 Codex、WorkBuddy 等主流 AI 工具。所有路径引用均为相对路径，无绝对路径依赖。

## 工作流程

### 1. 用户提问 → 先回答（文字结论）
- **默认只输出对话框文字结论**，不要自动生成文件
- 每条赛事必须包含以下**6项关键信息**：
  **赛事名称 / 双认证级别(田协+世界田联) / 比赛日期(精确到日) / 报名/出签日期 / 距用户交通耗时(12306实测) / 赛道难度备注(🟢🟡🔴含爬升)**
- 对话结束后**主动询问**是否需要整理成报告（HTML或PNG截图）

### 2. 报告 → 一次性输出完美成品
直接参考已生成的最终版HTML作为模板，关键点：
- **HTML结构必须完整闭合**，每个`<div class="it">`必须包含`in(m+tl)`+`r(st+c+d)`，不可残缺
- **所有赛事必须有全马/半马标签**（`<span class="t hf">全马</span>`）
- **所有赛事必须有爬升列**（右侧第2行`.c`）
- 不要用多次patch的方式修改HTML，DOM碎片化会导致样式错乱
- 建议一次性写完整文件，而非逐条edit

### 3. 报告4大板块结构
| 板块 | 标题 | 准入条件 | 图标规则 |
|------|------|---------|---------|
| 🎯 已报名 | 已报名 | 用户已报 | ✅已中签 / ⏳已报名 / ❌未中签 |
| 🔜 省外 | 蹲报名 · 省外 | A类及以上 | 无图标 |
| 🔜 省内 | 蹲报名 · 省内 | A类及以上 | 无图标 |
| 🏃 其他推荐 | 其他推荐赛道 | 非A但有特色 | 保留特色图标(👟💼🏞️等) |
- 省内/省外根据用户IP动态判断，禁止硬编码城市/省份名

### 4. 每条赛事卡片（从左到右）
```
[左侧]                              [右侧 3行]
赛事名称 [等级标签] [全马/半马]       [状态按钮]    ← 彩色圆角按钮
🚄 交通·费用·地点·日期               ⛰️~XXm       ← 灰色带颜色(🟢🟡🔴)
🏷️ 🟢/🟡 赛道特色一句话               日期         ← 灰色小字
```

### 5. HTML样式关键参数
- `.container max-width: 560px`（手机优先）
- `.it` 卡片：`display:flex`，`padding:10px 16px`
- `.r` 右侧列：`min-width:76px`，3行垂直排列
- 爬升颜色：`<80m=#059669🟢 / 80~200m=#d97706🟡 / 200m+=#dc2626🔴`
- 状态按钮顺序：已中签(深绿) > 已报名(浅绿) > 未中签(浅红) > 等报名(灰色)
- 所有条目结构必须完全一致，不允许有的带爬升有的不带

### 6. 采集 → 缺字段必须补
- 闹跑日期模糊（如"12月"）→ **必须搜最新新闻确认具体日**
- **撞期检查**：同省同日必须标注"⚠️ 撞期"
- 每条赛事12个必采字段（见 references/required_fields.md）

### 7. 认证查询（双体系）
- 田协 A/B/C 类：搜"2026年马拉松赛事目录 田协"
- 世界田联标牌：搜"2026世界田联标牌 中国 马拉松"
- **注意**：世界田联标牌≠田协A类，两者独立。如义乌半程是标牌（世界田联）+A类（田协），需同时标注

### 8. 交通查询
必须从 12306/携程 查真实数据，禁止凭距离估算。搜："出发城市到目标城市 高铁 最快"

### 9. 赛道爬升
搜"赛事名 赛道 爬升"，来源优先级：官方公布 > 跑友手表数据(佳明/高驰) > 媒体报告

### 10. 报告底部 → 数据来源
必须独立板块列出：爬升、认证级别、交通耗时的来源

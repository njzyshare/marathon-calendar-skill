# 马拉松个人赛历和推荐

## 跨工具兼容说明
本技能遵循标准 SKILL.md 格式，兼容 Codex、WorkBuddy 等主流 AI 工具。所有路径引用均为相对路径，无绝对路径依赖。

## 工作流程

### 1. 用户提问 → 先回答
- **默认只输出对话框文字结论**，不要自动生成文件
- 文字结论中每条赛事必须包含：**赛事名称 / 认证级别 / 比赛日期(精确到日) / 报名日期 / 出签日期 / 距用户交通耗时 / 难度备注(🟢🟡🔴)**
- 对话结束后**主动询问**是否需要整理成报告

### 2. 报告 → 询问格式
生成报告前必须问用户：是否生成？选 HTML 还是 PNG？

### 3. 报告底部 → 必须标注数据来源
每份报告底部必须独立板块标注数据来源。

### 4. 采集 → 缺字段必须补
- 闹跑日期如果是月份模糊（如"12月"），**必须搜最新新闻确认具体日**
- 每条赛事12个必采字段（见 references/required_fields.md）
- **撞期检查**：同省同日赛事必须标注"⚠️ 撞期"

### 5. 认证查询
- 田协 A/B/C 类：搜"2026年马拉松赛事目录 田协"
- 世界田联标牌：搜"2026世界田联标牌 中国 马拉松"

### 6. 交通查询
必须从 12306/携程 查真实数据，搜格式："出发城市到目标城市 高铁 最快"

### 7. 赛道备注
每条推荐必须附带爬升、景点、特色。搜："赛事名 赛道 爬升"

### 8. 报告HTML排版
- 手机优先：max-width 560px
- 每条赛事右侧3行：状态按钮→日期→爬升(带颜色)
- 状态按钮排序优先级：已中签 > 已报名 > 未中签
- 省内/省外：根据用户IP动态判断，禁止硬编码
- 数据来源底部独立列出

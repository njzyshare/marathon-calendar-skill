# 马拉松个人赛历和推荐

## 跨工具兼容说明
本技能遵循标准 SKILL.md 格式，兼容 Codex、WorkBuddy 等主流 AI 工具。所有路径引用均为相对路径，无绝对路径依赖。截图等工具调用通过上下文工作目录推导，不写死在技能文件中。

## 工作流程

### 1. 用户提问 → 先回答
- **默认只输出对话框文字结论**，不要自动生成文件
- 文字结论中每条赛事必须包含以下关键信息：
  **赛事名称 / 认证级别 / 比赛日期(精确到日) / 报名日期 / 出签日期 / 距用户交通耗时 / 难度备注(🟢🟡🔴)**
- 对话结束后**主动询问**是否需要整理成报告

### 2. 报告 → 询问格式
生成报告前必须问用户：是否生成？选 HTML 还是 PNG？

### 3. 报告底部 → 必须标注数据来源
每份报告（文字结论/HTML/PNG）底部必须标注每类信息的来源：
- 🏷️ **认证级别**：中国田协年度目录(A/B/C类) + 世界田联官网(标牌等级)
- 🏷️ **比赛日期/报名时间**：闹跑 nowrun.cn 详情页 + 最新新闻精确确认
- 🏷️ **交通耗时**：携程/12306 查询的最快直达高铁车次及二等座票价
- 🏷️ **赛道爬升/特色**：闹跑详情页 + 搜索引擎
- 🏷️ **赛事设项(全马/半马)**：闹跑详情页

文字结论中也要附简短的数据来源说明。HTML报告底部用独立板块列出。

### 4. 采集 → 缺字段必须补
- 闹跑日期如果是月份模糊（如"12月"），**必须搜最新新闻确认具体日**
- 每场赛事必须采集完整的关键字段（见 references/required_fields.md）
- **撞期检查**：同省同日赛事必须标注"⚠️ 撞期"

### 5. 认证查询
- 田协 A/B/C 类：搜"2026年马拉松赛事目录 田协"
- 世界田联标牌：搜"2026世界田联标牌 中国 马拉松"

### 6. 交通查询
必须从 12306/携程 查真实数据，禁止估算。
搜格式："出发城市到目标城市 高铁 最快"

### 7. 赛道备注
每条推荐必须附带备注：爬升数据、景点、特色、PB潜力。
搜索引擎搜 "赛事名 赛道 爬升"

### 8. 报告HTML排版
- 手机优先：max-width 560px，font-size 15px
- 卡片式：每个板块独立圆角卡片，白底阴影
- 等级标签：彩色圆角小标签，紧凑排列
- 状态列居右：报名状态+日期在右侧独立列
- 备注行：虚线分隔独立一行
- 数据来源：底部独立板块逐行列出

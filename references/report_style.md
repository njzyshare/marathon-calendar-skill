# 报告书写风格规范

## 整体结构

报告分为以下几个板块，按顺序排列：

1. **标题头** — 用户姓名 + 年份 + 主题
2. **排序说明** — 说明当前板块排序方式
3. **已报名板块** — 用户已报名的赛事，按状态排序：已中签(✅) > 已报名(⏳) > 未中签(❌)
4. **蹲省外板块** — 省外A类及以上赛事，按等级排序（无图标）
5. **蹲省内板块** — 省内A类及以上赛事，按等级排序（无图标）
6. **其他推荐板块** — 非A类但非常有特色的赛事（保留图标突出特色）
7. **底部声明** — 更新日期 + 数据来源

## 每条赛事卡片格式（HTML）

每条赛事用独立 `.it` 卡片，布局如下：

```
左侧：[图标] [赛事名称] [等级标签] [项目标签]
      🚄 Meta行：交通 · 费用 · 起终点 · 日期
      🏷️ 备注行：赛道特点一句话

右侧：[状态标签]    ← 第1行：凸出彩色按钮
      [日期]        ← 第2行：灰色小字
      [⛰️爬升]      ← 第3行：灰色小字，按爬升值带颜色
```

### Meta行规范
- 不超过4个要素，用`·`分隔
- 要素：交通(含车次) · 费用 · 地点 · 日期
- 日期格式：`11/1`（省略"周日"等冗余字眼）

### 备注行规范
- 控制在15字以内，一行完成
- 格式：`🏷️ 🟢/🟡 赛道特色亮点`
- 不包含"预计X月报名"等非赛道信息

### 右侧列（3行，顺序固定）
| 行 | 内容 | 样式 |
|---|------|------|
| 第1行 | 状态标签 | 彩色圆角按钮，视觉焦点 |
| 第2行 | ⛰️爬升 | 灰色小字 `font-size:10px`，按爬升数值带颜色 |
| 第3行 | 日期/预计报名时间 | 灰色小字 `color:#9ca3af` |

### 爬升颜色规则
| 爬升 | 颜色 | 样式 |
|------|------|------|
| <80m | 🟢 绿色 | `color:#059669` |
| 80~200m | 🟡 橙色 | `color:#d97706` |
| 200m+ | 🔴 红色 | `color:#dc2626` |
| 待补充 | 灰色 | `color:#9ca3af` |

### 等级标签一览
| 标签 | 含义 | 颜色指向 |
|------|------|---------|
| 💎白金标 | 世界田联最高 | 紫色背景 |
| 🥇金标 | 世界田联金标 | 金色背景 |
| 🏅精英标 | 世界田联精英 | 蓝色背景 |
| 🎖️标牌 | 世界田联标牌 | 靛蓝背景 |
| 🅰️A类 | 田协A类认证 | 绿色背景 |
| 📌必跑 | 强烈推荐 | 粉色背景 |
| 🏠超近 | 离家近 | 粉色背景 |

### 状态标签一览（按优先级排序）
| 标签 | 图标 | 颜色 |
|------|------|------|
| 已中签 | ✅ | 深绿底白字 |
| 已报名 | ⏳ | 浅绿底深绿字 |
| 未中签 | ❌ | 浅红底深红字 |
| 等报名 | — | 灰底深灰字 |

## 手机排版规范
- `max-width: 560px`
- `font-size: 15px; line-height: 1.5`
- 卡片式：圆角14px，白底阴影
- 右侧最小宽度76px

## 文字结论风格（对话框输出）

与报告风格一致但更精简，每条赛事控制在3行以内：

```
🏃 [赛事名称] [等级]
  📅 比赛：X月X日 · 📍 [城市] · 🚄 [交通耗时]
  🏷️ [赛道特色一句话]
  💬 [推荐理由或状态说明]
```

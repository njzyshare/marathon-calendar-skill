# 地区识别规则

## 用户IP位置获取
- 通过上下文 `<system-reminder>` 中的用户信息获取城市
- 不要写死在skill文件里
- 如果上下文没有明确城市，询问用户

## 基于位置的推荐加权
```
距用户 < 30min 高铁 → "🏠超近" 标签，自动加权推荐
距用户 < 1h 高铁    → "近" 标签
距用户 < 3h 高铁    → 正常推荐
距用户 > 6h 高铁    → 标注"建议飞机"
```

## 交通耗时查询方法
1. 用 `WebFetch` 或 `WebSearch` 搜索 "出发城市到目标城市 高铁 最快"
2. 优先使用携程/12306页面数据
3. 标注：最快耗时 + 车次号 + 二等座票价
4. 禁止凭距离和时间估算

## 省内/省外判断
- 省内赛事：举办城市与用户同省 → 单独归入"家门口"板块
- 省外赛事：按标牌等级排序，不按省份分类
